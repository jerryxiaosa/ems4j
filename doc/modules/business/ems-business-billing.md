# ems-business-billing 模块文档

## 1. 模块概述

`ems-business-billing` 是账务核心模块，负责余额管理、抄表消费、金额扣费、补正处理以及账务流水记录。

## 2. 核心功能

| 功能 | 说明 |
|------|------|
| 余额管理 | 账户/电表余额初始化、充值、扣费、查询、软删除 |
| 电量消费 | 抄表数据落库、分时电量差值计算、金额消费计算 |
| 账户消费 | 包月扣费、账户级消费记录 |
| 补正管理 | 电量补正、金额补正 |
| 账务流水 | 订单流水、余额变更、余额事件分发 |

## 3. 核心 Service

| Service | 职责 |
|---------|------|
| `BalanceService` | 余额初始化、充值、扣费、软删除 |
| `MeterConsumeService` | 抄表入库、消费计算、扣费触发 |
| `AccountConsumeService` | 账户级消费记录与查询 |
| `MeterCorrectionService` | 电量/金额补正 |

## 4. 模块边界

- 不再承载订单创建、支付回调、订单查询逻辑，这些能力已迁移到 `ems-business-order`
- 仍可通过 `OrderFlowEntity` 等账务流水对象记录余额与订单相关流水
- 对外提供账务能力给 `account`、`device`、`mq-rabbitmq` 使用

## 5. 当前实现说明

- 电表快照相关字段统一使用 `deviceNo`
- 余额表采用软删除与活跃唯一键策略，避免误操作历史余额记录
- 电表消费、账户消费和补正相关对外 DTO 均已迁移到 `billing` 包名
