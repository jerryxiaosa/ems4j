# ems-business-order 模块文档

## 1. 模块概述

`ems-business-order` 是交易与订单模块，负责订单创建、支付回调、订单查询、订单关闭与完成后的业务消息生成。

## 2. 核心功能

| 功能 | 说明 |
|------|------|
| 订单创建 | 按订单类型选择创建处理器，生成订单主表与明细 |
| 支付回调 | 处理第三方支付回调，校验金额并更新状态 |
| 订单查询 | 查询订单列表、订单详情 |
| 订单关闭/完结 | 主动关闭订单、检查支付完成、触发成功后置动作 |
| 三方支付 | 微信小程序支付、线下支付 |

## 3. 核心 Service

| Service | 职责 |
|---------|------|
| `OrderService` | 订单创建、关闭、支付回调与状态流转 |
| `OrderQueryService` | 订单列表与订单详情查询 |
| `ServiceRateService` | 服务费率读取 |

## 4. 处理器结构

- 创建处理器：`OrderCreationHandler`
- 完成处理器：`OrderCompletionHandler`
- 三方支付处理器：`OrderThirdPartyHandler`

当前能耗充值订单创建会在上层先做账户/电表业务校验，再进入订单模块保存快照与订单数据。

## 5. 模块边界

- 不直接依赖 `account` 与 `device` 业务模块，避免形成循环依赖
- 订单模块聚焦交易流程，不承载账户、电表主数据查询
- 成功后的业务后置动作通过 MQ 消息继续分发
